BlueML

Overview

Back in Fall of 2017, I joined BlueML when the project was barely a year old. Not only was I was intrigued by machine learning and web development, I felt that my work would actually be fulfilling as we have a direct impact on people in need by diagnosing cardiac and neurological disorders. Essentially the end goal would be for users to be able to enter EKG or EEG data, select features for classification, and receive an assessment with a high percentage of accuracy.

My Contributions

We had to completely revamp the previous implementation of the full stack and the machine learning algorithms as the basic looking UI/UX was using an outdated Bootstrap v2 and the accuracy was a mere ~50 percent accuracy for diagnosing arrhythmia. We first started off with EKG data as it is much simpler to interpret compared to EEG data. Thus, I researched features that can be interpreted from electrocardiogram (EKG) data to classify heart beats with machine learning and documented my findings. I also parsed the EKG files using Javascript into bitstream and converted the bitstream into JSON formats to be sent over to the server. On the frontend side I also worked on the overall look and functionality of the web app using HTML, CSS, and Bootstrap v4.0. I also got brief exposure to Angular and MongoDB as we prepare for further work over next semester.

Other Stuff
